Quy ước Đặt Tên Endpoint API Quản Lý Sách

1. Tài nguyên: Sách (/api/books)

HTTP Method Endpoint Chức năng
GET /api/books Lấy danh sách tất cả sách
GET /api/books/{id} Lấy chi tiết một cuốn sách theo ID
POST /api/books Thêm một cuốn sách mới
PUT /api/books/{id} Cập nhật thông tin sách theo ID
DELETE /api/books/{id} Xóa một cuốn sách theo ID

2. Tài nguyên: Tác giả (/api/authors)

HTTP Method Endpoint Chức năng
GET /api/authors Lấy danh sách tất cả tác giả
GET /api/authors/{id} Lấy thông tin chi tiết một tác giả
POST /api/authors Thêm mới một tác giả
PUT /api/authors/{id} Cập nhật thông tin tác giả
DELETE /api/authors/{id} Xóa tác giả theo ID

3. Tài nguyên: Thể loại (/api/categories)

HTTP Method Endpoint Chức năng
GET /api/categories Lấy danh sách thể loại
GET /api/categories/{id} Lấy chi tiết thể loại
POST /api/categories Thêm mới thể loại
PUT /api/categories/{id} Cập nhật thể loại
DELETE /api/categories/{id} Xóa thể loại

4. Quy tắc chung khi đặt tên endpoint

  • Không dùng động từ trong URL (động từ nằm ở HTTP method).
  • Plural form (dạng số nhiều): books, authors, categories.
  • Dùng `snake_case` hoặc `kebab-case` đồng nhất nếu cần thêm params: /api/books?author_id=1.
  • Không lồng nhiều cấp nếu không cần thiết. Ví dụ: /api/books/1/authors chỉ dùng khi thật sự có quan hệ lồng nhau cần xử lý.
